# Workflow: REVIEW_MODULE

## Proposito
Revisar un modulo existente para detectar mejoras tecnicas, visuales, responsive, seguridad y QA.

## Secuencia recomendada
1. Orchestrator define modulo y alcance.
2. Architect revisa estructura.
3. Backend Engineer revisa logica.
4. UI/UX Engineer revisa experiencia.
5. Mobile Responsive Engineer revisa mobile.
6. Security Engineer revisa riesgos.
7. QA Engineer define casos de prueba.
8. Documentation Engineer registra hallazgos.
9. Orchestrator prioriza cambios.

## Criterio de cierre
Informe con hallazgos, severidad, recomendaciones y proximo paso.
